# Changelog

Все заметные изменения проекта фиксируются в этом файле.

Формат версий:

- `1.0` — первый функциональный релиз
- `1.01`, `1.02`, `1.03` — исправления и небольшие UI/PWA-правки
- `1.1`, `1.2`, `1.3` — новая функциональность
- `2.0` — крупные изменения архитектуры

## 1.15

- Runtime-код перенесён напрямую в `index.html`, чтобы не зависеть от service worker injection.
- Кнопка «Обновить приложение» теперь находится в HTML приложения.
- Footer версии теперь находится в HTML приложения.
- Кнопка «Удалить сессию» теперь находится в HTML настроек сессии.
- Добавлена более надёжная логика сохранения состояния без зависимости от закэшированного runtime.
- Service worker постепенно возвращается к роли PWA/cache-слоя, а не источника UI-патчей.

## 1.14

- Исправлена проблема со старыми injected runtime-скриптами внутри закэшированного HTML.
- Service worker теперь принудительно удаляет старые runtime/persistence patch scripts перед вставкой новых.
- Исправлено исчезновение кнопки «Обновить приложение» после обновлений.
- Исправлено исчезновение footer версии.
- Исправлено отсутствие кнопки «Удалить сессию» после обновления PWA.
- Улучшена система обновления runtime-патчей внутри PWA.

## 1.13

- Добавлена кнопка «Удалить сессию» в настройках сессии.
- Удаление оформлено в iOS-style destructive action.
- После удаления пользователь автоматически возвращается на экран «История».
- История и статистика автоматически обновляются после удаления.
- Состояние приложения сохраняется сразу после удаления сессии.

## 1.12

- Исправлено повторное применение сохранённого состояния после каждого клика.
- Исправлено открытие старой сессии после создания новой.
- Исправлено зависание интерфейса после обновления приложения.
- Runtime persistence теперь применяется только один раз при старте приложения.
- Улучшено поведение обновления PWA и восстановления состояния.

## 1.11

- Исправлен конфликт runtime patch / persistence patch в service worker.
- Кнопка «Обновить приложение» больше не должна пропадать после обновления.
- Footer версии больше не должен исчезать после обновления.
- После обновления приложение возвращается в раздел «Настройки», чтобы сразу проверить текущую версию.

## 1.10

- Исправлено сохранение истории тренировок после перезапуска PWA.
- Исправлено восстановление пустой истории: если история очищена, демо-сессии больше не должны возвращаться.
- Добавлено сохранение истории и экипировки в localStorage.
- Данные сохраняются перед обновлением приложения, при закрытии PWA и периодически во время работы.

## 1.09

- Восстановлена кнопка «Обновить приложение» после ошибочного упрощения sw.js.
- Восстановлен footer версии в настройках.
- Версия в настройках обновлена до 1.09.

## 1.08

- Добавлен footer версии внизу раздела «Настройки».
- Добавлен текст версии приложения.
- Добавлен copyright footer.

## Ранние PWA/UI fixes

- Добавлена PWA-структура: manifest, service worker, iOS meta, icons.
- Добавлена поддержка iOS safe area, чтобы интерфейс не уезжал под Dynamic Island/status bar.
- Отключено double tap zoom / pinch zoom в iOS PWA.
- Добавлена кнопка «Обновить приложение».
- Добавлен iOS-style overlay со спиннером при обновлении приложения.
